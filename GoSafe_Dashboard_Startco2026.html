Railway — compartir con socios

Docker + Node estático — sin API keys en el repo
DEPLOY
En la raíz del repo: Dockerfile, server.mjs, railway.toml. El contenedor copia GoSafe_Dashboard_Startco2026.html a public/index.html y sirve en PORT (Railway lo inyecta).
Railway: New project → Deploy from GitHub (o CLI) → root Startco26 → generar dominio público → compartir URL. Variables en Railway (no en el repo): MAILERLITE_API_KEY o MAILES_API_KEY, CALENDLY_URL, OUTREACH_API_SECRET, opcional MAILERLITE_GROUP_NAME, opcional PUBLIC_BASE_URL (URL pública del deploy, sin barra final; el logo del mail usa /go-safe-logo.png y así evitas base64 en el HTML).
railway.app →

MailerLite — flujo técnico

API token en .env como MAILES_API_KEY (o MAILERLITE_API_KEY)
ENVIO
Plantilla HTML: gosafe_vc_outreach_email.html (merge {$name} + enlace Calendly). Vista previa (mismo servidor): /email-preview?name=Carolina — cambia ?name= para ver el saludo.
Script: send_gosafe_vc_outreach.py crea/reutiliza el grupo GoSafe VC Outreach (Startco 2026), da de alta al suscriptor, arma la campaña y programa envío instant.
Calendly: define CALENDLY_URL en .env con tu enlace público (15 min).
Prueba tipo VC: /usr/bin/python3 send_gosafe_vc_outreach.py --email caromontefrio@gmail.com --name "Carolina" — --dry-run imprime el HTML sin llamar a la API.
Si la campaña queda en estado ready en el panel de MailerLite, revisa autenticación de dominio y confirma el envío manual si la cuenta lo pide. Panel: dashboard.mailerlite.com
Enviar desde el sitio desplegado (mismo dominio Railway): el servidor expone POST /api/send-outreach. La API key de MailerLite nunca va al navegador; solo quien tenga el OUTREACH_API_SECRET configurado en Railway puede disparar.
